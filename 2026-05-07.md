# LangGraph 에이전트 기초 정리

## 1. LangGraph란?

LangGraph는 LLM(대규모 언어모델)을 단순 챗봇이 아니라  
"상태(State)를 가진 업무 프로세스 시스템"처럼 만들 수 있게 해주는 프레임워크다.

기존 GPT 방식:

```text
질문 → GPT → 답변
```

LangGraph 방식:

```text
질문
 ↓
생각
 ↓
도구 사용
 ↓
메모리 확인
 ↓
다시 판단
 ↓
최종 답변
```

즉:

- 스스로 판단
- 필요한 도구 사용
- 기억 유지
- 여러 단계 작업 수행

이 가능한 Agent 시스템을 만드는 것이 목적이다.

---

# 2. LangGraph 핵심 구성 요소

## 2-1. State (상태)

현재 상태나 데이터를 저장하는 공간.

예시 코드:

```python
class State(TypedDict):
    messages: list
```

의미:

```text
현재까지의 대화 저장
```

예시 데이터:

```python
{
  "messages": [
    "안녕",
    "오늘 날씨 알려줘"
  ]
}
```

### 쉽게 이해하는 예시

배달앱 상태:

```text
- 메뉴 선택 완료
- 주소 입력 완료
- 결제 전
```

이런 현재 상태 저장소가 State이다.

---

## 2-2. Node (작업)

실제 행동을 수행하는 함수.

예시:

```python
def chatbot(state):
    return {"messages": [llm.invoke(state["messages"])]}
```

의미:

```text
현재 대화를 기반으로
LLM이 답변 생성
```

### 쉽게 이해하는 예시

회사 조직으로 생각하면:

| Node | 역할 |
|---|---|
| 검색 Node | 자료 검색 |
| 요약 Node | 내용 정리 |
| 검증 Node | 사실 확인 |
| 답변 Node | 사용자 응답 |

즉:

```text
Node = 일을 수행하는 직원
```

---

## 2-3. Edge (연결 흐름)

Node 실행 후 다음 어디로 이동할지 정의.

예시:

```python
graph.add_edge("chatbot", END)
```

의미:

```text
chatbot 작업 후 종료
```

### 흐름 예시

```text
사용자 질문
   ↓
검색
   ↓
요약
   ↓
답변
```

이 연결 자체가 Edge이다.

---

## 2-4. Graph (전체 구조)

전체 흐름을 구성.

예시:

```python
graph = StateGraph(State)

graph.add_node("chatbot", chatbot)

graph.add_edge(START, "chatbot")
graph.add_edge("chatbot", END)
```

구조:

```text
START
  ↓
chatbot
  ↓
END
```

---

## 2-5. compile()

```python
graph.compile()
```

의미:

```text
실행 가능한 형태로 변환
```

웹개발의 build 느낌과 유사하다.

---

## 2-6. invoke()

```python
graph.invoke(...)
```

의미:

```text
실제 실행
```

# 3. 첫 번째 노트북 핵심 요약

```text
State = 기억
Node = 행동
Edge = 흐름
Graph = 전체 프로세스
```

즉:

> LLM을 하나의 업무 프로세스로 만드는 구조

---

# 4. Tool(도구) 개념

LLM은 기본적으로:

- 실시간 검색 불가
- 계산 약함
- DB 조회 불가

그래서 외부 기능을 연결한다.

예시:

```python
@tool
def search_weather(city: str):
    return f"{city}는 맑음"
```

---

## Tool 사용 흐름

사용자:

```text
서울 날씨 알려줘
```

Agent 내부:

```text
1. 날씨 정보 필요 판단
2. weather tool 호출
3. 결과 수신
4. 답변 생성
```

---

## 현실 예시

| Tool | 역할 |
|---|---|
| 웹검색 | 최신 정보 검색 |
| 계산기 | 숫자 계산 |
| DB조회 | 고객 정보 조회 |
| API 호출 | 외부 시스템 연동 |

---

# 5. Memory(메모리)

Agent가 이전 내용을 기억하도록 만드는 기능.

기존 GPT:

```text
질문 → 답변 → 종료
```

Agent:

```text
이전 대화 기억
사용자 정보 기억
중간 결과 저장
```

---

## 예시

사용자:

```text
내 이름은 지혜야
```

몇 분 후:

```text
내 이름 뭐야?
```

메모리 없을 때:

```text
모름
```

메모리 있을 때:

```text
지혜입니다
```

---

# 6. Checkpointer

대화 상태를 저장하는 기능.

예시:

```python
MemorySaver()
```

의미:

```text
현재 대화 상태 저장
```

---

# 7. thread_id

사용자별 대화 구분 ID.

예시:

```python
config = {
  "configurable": {
    "thread_id": "user1"
  }
}
```

의미:

```text
사용자별 대화 분리
```

---

# 8. 실무 관점에서 중요한 이유

LangGraph가 실무에서 중요한 이유:

| 기능 | 의미 |
|---|---|
| 상태 관리 | 긴 작업 가능 |
| 흐름 제어 | 복잡한 분기 처리 |
| 메모리 | 사용자 기억 |
| Tool 연결 | 실제 액션 수행 |
| Multi-Agent | 여러 AI 협업 가능 |

즉:

```text
회사 업무 프로세스를 AI화하기 쉬움
```

# 9. 앞으로 만들 수 있는 수준

## Level 1 — 단순 챗봇

```text
질문 → 답변
```

---

## Level 2 — 검색 Agent

```text
질문
→ 웹검색
→ 요약
→ 답변
```

---

## Level 3 — 업무 자동화

```text
메일 확인
→ 요약
→ 중요도 판단
→ 슬랙 전송
```

---

## Level 4 — Multi Agent 시스템

```text
기획 AI
개발 AI
검토 AI
```

여러 AI가 협업 가능.

# 10. 최종 핵심 요약

LangGraph 핵심 4가지:

```text
1. State = 기억
2. Node = 행동
3. Edge = 흐름
4. Tool = 외부 기능
```

한 줄 요약:

> LangGraph는 LLM을 상태(State)를 가진 업무 프로세스 시스템으로 만드는 프레임워크이다.
