0. 먼저 큰 그림부터

기존 GPT 챗봇은 보통 이런 구조였어.

질문 → GPT → 답변

근데 AI Agent는 다름.

질문
 ↓
생각
 ↓
도구 사용
 ↓
메모리 확인
 ↓
다시 판단
 ↓
최종 답변

즉:

스스로 판단하고
필요한 도구를 쓰고
기억도 하고
여러 단계로 일하는 AI

를 만드는 거야.

LangGraph는 이 과정을 “그래프(흐름도)”처럼 만드는 프레임워크임.

1. LangGraph 핵심 개념

LangGraph에서 제일 중요한 건 4개.

개념	의미
State	현재 상태(메모리)
Node	작업 수행
Edge	다음 이동 경로
Graph	전체 흐름

쉽게 말하면:

State = 현재 상황 저장소
Node = 직원
Edge = 직원끼리 연결선
Graph = 회사 업무 흐름도
2. 첫 번째 노트북 — LangGraph 에이전트 기초

이 노트북은:

“AI가 단계적으로 일하게 만드는 법”

을 배우는 단계야.

2-1. State (상태)

보통 이렇게 만듦.

class State(TypedDict):
    messages: list

의미:

현재 대화 내용을 저장

예시:

{
  "messages": [
    "안녕",
    "오늘 날씨 알려줘"
  ]
}

즉 AI가 기억할 데이터 공간.

쉽게 이해하는 예시
배달앱 예시
현재 주문 상태:
- 메뉴 선택됨
- 주소 입력됨
- 결제 안함

이게 State야.

AI Agent도 똑같음.

2-2. Node (작업)

Node는 실제 행동하는 함수.

예시:

def chatbot(state):
    return {"messages": [llm.invoke(state["messages"])]}

의미:

현재 대화를 보고
LLM이 답변 생성
쉽게 이해하는 예시

회사로 치면:

Node	역할
검색 Node	자료 찾기
요약 Node	내용 요약
검증 Node	사실 확인
답변 Node	사용자 응답

즉 Node = 직원.

2-3. Edge (이동 경로)

Node 끝나면 어디로 갈지 결정.

graph.add_edge("chatbot", END)

뜻:

chatbot 끝나면 종료
예시
사용자 질문
   ↓
검색
   ↓
정리
   ↓
답변

이 흐름 자체가 Edge.

2-4. Graph 만들기

전체 연결.

graph = StateGraph(State)

graph.add_node("chatbot", chatbot)

graph.add_edge(START, "chatbot")
graph.add_edge("chatbot", END)
그림으로 보면
START
  ↓
chatbot
  ↓
END
2-5. compile()
graph.compile()

이건:

실행 가능한 앱으로 변환

하는 단계.

웹개발로 치면 build 느낌.

2-6. invoke()
graph.invoke(...)

실제 실행.

여기까지 핵심 요약

첫 번째 노트북 핵심:

State = 기억
Node = 행동
Edge = 흐름
Graph = 전체 프로세스

즉:

“LLM을 하나의 업무 프로세스로 만든다”

가 핵심.

3. 두 번째 노트북 — 도구 + 메모리

여기서부터 진짜 Agent 느낌 남.

3-1. Tool (도구)

LLM은 원래 계산/검색 못함.

그래서 Tool 붙임.

예시:

@tool
def search_weather(city: str):
    return f"{city}는 맑음"
왜 필요함?

GPT는 원래:

❌ 실시간 검색 못함
❌ DB 조회 못함
❌ 계산 약함

그래서 외부 기능 연결.

현실 예시
Tool	역할
웹검색	최신 정보 검색
계산기	숫자 계산
DB조회	고객 정보 조회
API호출	외부 서비스 연결
3-2. Tool Calling

AI가 스스로 판단함.

예:

사용자:
서울 날씨 알려줘

AI 내부:

1. 날씨 필요하네?
2. weather tool 호출
3. 결과 가져오기
4. 답변 생성

이게 Agent의 핵심.

쉽게 이해하는 예시

비서 AI라고 생각하면 쉬움.

사장:
오늘 회의 일정 알려줘

비서:
(캘린더 조회)
→ 일정 확인
→ 응답
3-3. Memory (메모리)

이게 엄청 중요함.

기존 GPT:

질문 → 답
끝

Agent:

과거 대화 기억
사용자 성향 기억
중간 결과 저장
예시

사용자:

내 이름은 지혜야

몇 분 뒤:

내 이름 뭐였지?

메모리 없으면:

몰라요

메모리 있으면:

지혜입니다
3-4. Checkpointer

LangGraph에서 메모리 저장장치.

예시:

MemorySaver()

의미:

대화 상태 저장
3-5. thread_id

대화 구분 ID.

예:

config = {
  "configurable": {
    "thread_id": "user1"
  }
}
왜 필요?

카톡 생각하면 쉬움.

A 사용자 대화
B 사용자 대화

섞이면 안 되잖아.

그래서 thread_id로 구분.

4. 이 노트북들의 진짜 핵심

이 두 개가 결국 말하는 건:

LLM을 단순 채팅이 아니라
"업무 수행 시스템"으로 만들자

임.

5. 진짜 중요한 포인트 (실무)

LangGraph가 인기인 이유:

특징	의미
상태 관리	긴 작업 가능
흐름 제어	복잡한 분기 가능
메모리	사용자 기억
Tool 연결	실제 액션 가능
Multi-Agent	여러 AI 협업

즉:

"회사 업무 프로세스"

를 AI로 만들기 쉬움.

6. 네가 앞으로 만들게 될 수준

이거 이해하면 다음 가능.

수준 1 — 챗봇
질문 → 답변
수준 2 — 검색 Agent
질문
→ 웹검색
→ 정리
→ 답변
수준 3 — 업무 자동화
메일 확인
→ 요약
→ 중요도 판단
→ 슬랙 전송
수준 4 — Multi Agent
기획 AI
개발 AI
검토 AI

서로 협업.

7. 한 줄로 요약

LangGraph는:

"LLM을 상태(State)를 가진 업무 프로세스 시스템으로 만드는 프레임워크"

야.

8. 네가 지금 꼭 이해해야 하는 최소 핵심

이 4개만 이해하면 절반 성공.

1. State = 기억
2. Node = 행동
3. Edge = 흐름
4. Tool = 외부 기능
